## UX Writing Check Examples (OK / NG)

### 1. Avoid Redundant Expressions

**Focus**: Keep it short and direct

*   ✅ OK
    *   Download the file
    *   Add a user
    *   Settings have been saved

*   ❌ NG
    *   You are able to download the file
    *   Perform user addition
    *   The saving of settings has been completed

***

### 2. Avoid Excessive Politeness

**Focus**: Prioritize readability over overly formal language

*   ✅ OK
    *   Please check the details
    *   Enter your password
    *   Registration is complete

*   ❌ NG
    *   We kindly ask that you please review the details
    *   We humbly request that you enter your password
    *   We have completed your registration for you

***

### 3. Do Not Repeat the Same Meaning

**Focus**: One message = one idea

*   ✅ OK
    *   Failed to save. Please try again
    *   The email address is invalid

*   ❌ NG
    *   Failed to save. The save process did not complete successfully, so please try again
    *   The email address is invalid. Please enter a valid email address  
        *(The second sentence repeats the same meaning)*

***

### 4. Do Not Omit Particles (Clarify Sentence Structure)

**Focus**: Make relationships in the sentence clear

*   ✅ OK
    *   The file has been deleted
    *   Please change the settings
    *   Login completed

*   ❌ NG
    *   File deleted
    *   Change settings please
    *   Login completed successfully

***

### 5. Avoid Internal Jargon and Technical Terms

**Focus**: Use words that anyone can understand

*   ✅ OK
    *   Working folder
    *   Selected page
    *   Save the image

*   ❌ NG
    *   Current directory
    *   Active page context
    *   Persist the asset

***

## Combined Example Covering All Points

### ✅ Good Example (Meets All Criteria)

*   Delete this account. Are you sure?
*   Please check your input
*   The file has been uploaded
*   The password is incorrect

👉 Characteristics

*   Short
*   Not overly polite
*   No duplication
*   Clear sentence structure
*   Easy to understand

***

### ❌ Bad Example (Multiple Issues)

*   We will proceed with executing the deletion of your account. Are you sure?
*   We kindly ask that you please review your input
*   The file upload process has been carried out
*   There is a possibility that the password is incorrect, so please check

👉 Issues

*   Verbose
*   Overly polite
*   Indirect
*   Ambiguous tone

***

## Ready-to-Use Checklist

Use the following checklist during reviews:

*   Is the sentence too long? (e.g., “you are able to…”)
*   Is the tone too polite or formal?
*   Is the same idea repeated twice?
*   Is the sentence structure clear?
*   Are the words understandable for general users?
