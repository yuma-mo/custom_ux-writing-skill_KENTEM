## ✅ Good Example (Following the Rules)

The following sentences correctly use each type of symbol.

There are two types of permissions in the management function: “Administrator” and “Super User.”  
Enter “1000” in the “Username” field and click \[Register].  
Updating the data to match the state of the base system is called “data synchronization.”  
Delete 【Construction Data for FY2024】.  
The connection URL is {subdomain}.kentem.jp.  
Please set a password (at least 8 characters).  
Example: Set a password such as “abc12345”.

👉 Key Points

*   （） = supplementary explanation
*   “” = input values, emphasis, definitions, or terms
*   ［］ = buttons or UI item names
*   【】 = user-defined names
*   {} = variables
*   Use “Example:” for examples

***

## ❌ Bad Example (Violating the Rules)

The following intentionally uses incorrect symbols for the same content.

There are two types of permissions in the management function: (Administrator) (Super User).  
Enter (1000) in the “Username” field and click “Register.”  
Updating the data to match the state of the base system is called \[data synchronization].  
Delete (Construction Data for FY2024).  
The connection URL is \[subdomain].kentem.jp.  
Please set a password “at least 8 characters.”  
(Example) Set a password such as abc12345.

👉 Issues

*   Using （） where “” should be used
*   Using “” instead of ［］ for operation buttons
*   Using ［］ for defined terms (incorrect purpose)
*   Expressing user-defined names with （）
*   Using ［］ for variables
*   Using “” for supplementary explanations
*   Using “(Example)” instead of “Example:”

***

## Summary (Checklist)

If all the following criteria are met, it is correct:

*   Supplementary information → （）
*   User input / emphasis / terms → “”
*   UI elements (buttons, fields) → ［］
*   User-defined names → 【】
*   Variables → {}
*   Examples → “Example:”
