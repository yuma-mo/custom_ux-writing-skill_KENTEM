## Scope

This rule applies only to the following UI components.
* button

***

## ✅ Good Examples (Meet All Rules)

The following are button label examples that satisfy all evaluation criteria.

#### Pattern ①: Verb (dictionary form)

*   Import
*   Cancel
*   Go back

👉 **Key Points**

*   Written in the dictionary (base) form of the verb

#### Pattern ②: Verbal nouns (omit “do”)

*   Add
*   Delete

👉 **Key Points**

*   Do not include “do” (e.g., avoid “add do” / “delete do”)

#### Pattern ③: Object + Verb (with particle “wo” equivalent → direct object)

*   Add permissions
*   Cancel request

👉 **Key Points**

*   Use verbs in dictionary form
*   Clearly specify the object
*   Use a direct object structure (equivalent to “を” in Japanese)

***

## ❌ Bad Examples (Covering All Rule Violations)

#### Pattern ①: Not in dictionary form

*   Please add
*   Please delete

👉 **Issues**

*   Uses polite/request forms instead of dictionary form

#### Pattern ②: Includes unnecessary “do”

*   Add do
*   Delete do

👉 **Issues**

*   Incorrectly keeps the “do” in verbal noun expressions

#### Pattern ③: Noun-only expressions

*   Addition
*   Cancellation

👉 **Issues**

*   Not expressed as verbs (even when intended as actions)

#### Pattern ④: Incorrect particle usage (equivalent to “no”)

*   Addition of permissions
*   Cancellation of request

👉 **Issues**

*   Uses a noun phrase instead of an action-oriented label

#### Pattern ⑤: Unnecessarily long / verbose

*   Add permissions do
*   Cancel the request do

👉 **Issues**

*   Includes unnecessary words
*   Lacks simplicity

***

## ✅ Summary (Checklist)

Good button labels should satisfy the following:

*   ✅ Use the verb in dictionary (base) form
*   ✅ Omit “do” in verbal noun expressions
*   ✅ Use a direct object when needed
*   ✅ Keep it short and simple
